# Codex WebSocket Integration

Real-time streaming of Codex CLI output to external applications via WebSocket.

## How it works

1. The Rust TUI (`chatwidget.rs`) broadcasts all streamed text to a WebSocket server
2. Python clients connect and receive the output in real-time
3. The text is streamed character-by-character as it's generated by the LLM

## Setup

### 1. Enable WebSocket Streaming in Codex TUI

The WebSocket server is embedded in the Codex TUI. To enable it, add this call to `chatwidget.rs` initialization:

```rust
// In ChatWidget::new() or in the app initialization
widget.enable_websocket_streaming("127.0.0.1:8765".to_string())?;
```

### 2. Start Codex

```bash
cd /Users/wiggel/IntelliJIDEA/codex_clone/codex
./bazel-bin/codex-rs/cli/codex
```

The WebSocket server will start automatically and listen on `ws://127.0.0.1:8765`.

### 3. Run the Python Client

**Using uv (recommended):**
```bash
cd codex-rs/tui
uv venv .venv
uv pip install websockets
.venv/bin/python python_websocket_client.py
```

**Using traditional venv:**
```bash
cd codex-rs/tui
python -m venv codex_ws_client
source codex_ws_client/bin/activate  # On Windows: codex_ws_client\Scripts\activate
pip install websockets
python python_websocket_client.py
```

**Using pipx for isolated installation:**
```bash
pipx runpip websockets
pipx run python_websocket_client.py --uri ws://127.0.0.1:8765
```

## Configuration

### Server-side (Rust)

```rust
// Custom bind address
widget.enable_websocket_streaming("0.0.0.0:8765".to_string())?;

// Or for external access
widget.enable_websocket_streaming("0.0.0.0:8765".to_string())?;
```

### Client-side (Python)

```bash
# Custom URI
python python_websocket_client.py --uri ws://192.168.1.100:8765

# No auto-reconnect
python python_websocket_client.py --no-reconnect

# Custom reconnect delay
python python_websocket_client.py --reconnect-delay 5
```

## API

### WebSocket Message Format

The server sends plain text messages:

```
H
e
l
l
o

w
o
r
l
d
!
```

Each message contains a chunk of text (could be a single character or multiple characters).

### Example Client in Python

```python
import asyncio
import websockets

async def stream_codex():
    uri = "ws://127.0.0.1:8765"
    async with websockets.connect(uri) as ws:
        async for message in ws:
            text = message if isinstance(message, str) else message.decode('utf-8')
            print(text, end='', flush=True)
            # Process the text in your application
            await my_send_to_app(text)

asyncio.run(stream_codex())
```

### Example Client in JavaScript

```javascript
const ws = new WebSocket('ws://127.0.0.1:8765');

ws.onopen = () => {
    console.log('Connected to Codex WebSocket');
};

ws.onmessage = (event) => {
    const text = event.data;
    process.stdout.write(text);
    // Or: document.getElementById('output').textContent += text;
};

ws.onclose = () => {
    console.log('Codex WebSocket connection closed');
};
```

## Troubleshooting

### "Could not connect to ws://127.0.0.1:8765"
- Make sure Codex is running
- Check if WebSocket streaming is enabled in the code
- Verify the bind address matches

### No output appears
- Check if the Codex TUI is actually generating output (try typing something in Codex)
- Look for "WebSocket client connected" message in Codex terminal

### Connection drops frequently
- This is normal if Codex is restarted - the client should auto-reconnect
- Check network stability for remote connections
